JSON
 4. Создать внешний репозиторий c названием JSON.                                                              add new repository "JSON"
 5. Клонировать репозиторий JSON на локальный компьютер.                                                       git clone JSON https://github.com/Antontest01/JSON.git
 6. Внутри локального JSON создать файл “new.json”.                                                            cd JSON && touch file1.json
 7. Добавить файл под гит.                                                                                     git add .
 8. Закоммитить файл.                                                                                          git commit -m "first"
 9. Отправить файл на внешний GitHub репозиторий.                                                              git push
 10. Отредактировать содержание файла “new.json” - написать информацию о себе                                  vim fil1.json
(ФИО, возраст, количество домашних животных, будущая желаемая зарплата). Всё написать в формате JSON.
 11. Отправить изменения на внешний репозиторий.                                                               git add . && git commit -m "second" && git push
 12. Создать файл preferences.json                                                                             touch preferens.json
 13. В файл preferences.json добавить информацию о своих предпочтениях                                         vim preferences.json
(Любимый фильм, любимый сериал, любимая еда, любимое время года, сторона которую хотели бы посетить) в формате JSON. vim preferans.json
 14. Создать файл sklls.json добавить информацию о скиллах которые будут изучены на курсе в формате JSON       vim skills.json
 15. Отправить сразу 2 файла на внешний репозиторий.                                                           git add . && git commit | git push                               
 16. На веб интерфейсе создать файл bug_report.json.                                                           add new file/ name-bug_report.json
 17. Сделать Commit changes (сохранить) изменения на веб интерфейсе.                                           commit changes
 18. На веб интерфейсе модифицировать файл bug_report.json, добавить баг репорт в формате JSON.                {"number":1,2,3} | 
 19. Сделать Commit changes (сохранить) изменения на веб интерфейсе.                                           changes commit -m "1"
 20. Синхронизировать внешний и локальный репозиторий JSON                                                     git pull
